### 최단 경로 알고리즘

- 최단 경로 알고리즘은 말 그래도 가장 짧은 거리를 찾는 알고리즘으로 크게 세 가지 상황을 고려해볼 수 있다.
1. 한 지점에서 다른 모든 지점으로의 최단 경로를 구할 때 : 다익스르타 알고리즘 사용
2. 모든 지점에서 다른 모든 지점으로의 최단 경로를 구할 때 : 플로이드 워셜 알고리즘 사용.
3. 음의 간선이 존재할 때 : 벨만 포드 알고리즘 사용

- 각각 알고리즘이 필요할 떄가 다르고 시간 복잡도 또한 크게 차이가 나므로 주어진 조건을 잘 고려하여 위 세가지 알고리즘 중 하나를 선택해야 한다.

### 다익스트라 알고리즘

- 다익스트라 알고리즘은 특정 노드에서 출발하여 모든 노드로의 최단 거리를 구할 때 사용한다.
- 사용 조건으로는 음의 간선이 없을 때 사용 가능하다.
- 다익 스트라 알고리즘은 그리디 알고리즘으로 분류 된다. 이유: 매 과정에서 '가장 적은 비용의 노드' 를 선택하는 과정이 포함됨.
- 다익 스트라 작동 원리

1. 출발 노드 설정
2. 최단 거리 테이블 초기화
3. 방문하지 않은 노드 중에서 최단 거리가 가장 짧은 노드 선택.
4. 해당 노드를 거쳐 다른 노드로 가는 비용을 계산하여 최단 거리 테이블 갱신.
5. 3-4번을 반복

- 다익스트라 알고리즘은 대표적으로 두 개의 시간복잡도를 갖는 알고리즘이 존재.
- 첫번째는 O(V^2) , 두번째는 O(ElogV)
- 두 알고리즘의 차이는 매번 가장 적은 비용의 노드를 선택할 때 첫번째 알고리즘의 경우에는 순차탐색을 하고 두번째 알고리즘은 최소힙으로 선택을 한다는 점이다.

- O(ElogV)알고리즘
    - 우선순위 큐를 이용하여 구현 >> PriorityQueue 또는 heapq 라이브러리 사용
    - 파이썬 라이브러리는 기본적으로 최소 힙 구조를 이용
    - 힙 삽입 : O(logN) 삭제 : O(logN)
    

### 플로이드 워셜 알고리즘

- 플로이드 워셜 알고리즘의 경우에는 모든 지점에서 다른 모든 지점까지의 최단 경로를 구해야 하는 경우에 사용한다.
- 다이나믹 프로그래밍을 사용함 / O(v^3)의 시간 복잡도를 갖는 알고리즘.
